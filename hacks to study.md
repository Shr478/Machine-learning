Here are **5 practical hacks** to **study and memorize lab programs effectively** using **pen, paper, and a compiler**. These methods help you remember logic instead of just mugging up code. ✍️💻

---

## 1️⃣ Break the Program into 4 Small Parts

Instead of memorizing the whole code, divide it into **4 sections**:

1. **Header & libraries**
2. **Variable declarations**
3. **Core logic (loops / conditions)**
4. **Output / printing**

Example structure:

```
#include<stdio.h>

int main() {
   // variables

   // logic

   // output
}
```

🧠 **Hack:** Memorize the **structure first**, then fill in the logic.

---

## 2️⃣ Write the Logic in Plain English First

Before coding, write the **algorithm in simple steps** on paper.

Example:

Program: Factorial

```
1. Read number n
2. Set fact = 1
3. Loop from 1 to n
4. Multiply fact with i
5. Print fact
```

🧠 When you remember the **steps**, the code becomes easy to reconstruct.

---

## 3️⃣ The 3-Time Writing Rule

This works very well for lab exams.

1. **Read the program once**
2. **Write it on paper without seeing**
3. **Compile and check errors**
4. **Correct mistakes**
5. **Write again**

✍️ Writing **3 times** locks the program into memory.

---

## 4️⃣ Dry Run with Pen & Table

Use a **dry run table** for loops and conditions.

Example:

| i | fact |
| - | ---- |
| 1 | 1    |
| 2 | 2    |
| 3 | 6    |
| 4 | 24   |

🧠 This helps you **understand how the program works**, so you don’t forget the logic.

---

## 5️⃣ Practice the “Blank Page Method”

This is the **most powerful hack**.

1. Look at program for **2 minutes**
2. Close the book
3. Write it **from memory on a blank page**
4. Compile and test

If wrong → correct → repeat.

📌 After **3–4 repetitions**, you will remember it permanently.

---

✅ **Bonus Tip (Very Powerful):**
Create a **program template** you memorize once:

```
#include<stdio.h>

int main() {
    // variable declaration

    // input

    // logic

    // output

    return 0;
}
```

Most C lab programs fit inside this structure.
